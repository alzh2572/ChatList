# План реализации ChatList

Поэтапный порядок работ по реализации программы на основании [PROJECT.md](PROJECT.md).

---

## Этап 1. Подготовка проекта

1. Обновить `requirements.txt`: PyQt6, httpx (или requests), python-dotenv.
2. Создать структуру модулей:
   - `main.py` — точка входа и GUI
   - `db.py` — работа с SQLite
   - `models.py` — логика моделей нейросетей
   - `network.py` — HTTP-запросы к API
3. Добавить `.env.example` с примерами переменных для API-ключей.
4. Убедиться, что `.env` указан в `.gitignore`.

---

## Этап 2. База данных (`db.py`)

1. Реализовать инициализацию БД: создание файла SQLite и таблиц по схеме из [DATABASE.md](DATABASE.md).
2. Добавить CRUD-функции для таблицы `prompts` (создание, чтение, поиск, удаление).
3. Добавить CRUD-функции для таблицы `models` (создание, чтение, обновление, удаление, фильтр активных).
4. Добавить CRUD-функции для таблицы `results` (сохранение выбранных строк, чтение истории).
5. Добавить функции для таблицы `settings` (получение и запись настроек по ключу).
6. Написать начальные данные (seed): одна-две тестовые модели, если таблица пуста.

---

## Этап 3. Работа с API (`network.py`, `models.py`)

1. В `models.py` описать структуру данных модели (имя, URL, имя переменной ключа, активность).
2. Реализовать загрузку API-ключей из `.env` по имени переменной из поля `api_id`.
3. В `network.py` реализовать базовую функцию отправки промта в одну модель (POST-запрос, обработка ошибок, таймаут).
4. Добавить поддержку разных типов API (OpenAI-совместимый формат как базовый; при необходимости — адаптеры для DeepSeek, Groq и др.).
5. Реализовать параллельную или последовательную отправку промта во все активные модели.
6. Вернуть список результатов в едином формате: `{model_name, response_text, error?}`.

---

## Этап 4. Временная таблица результатов (в памяти)

1. Создать класс или структуру для хранения текущей сессии запроса (промт + список ответов + флаг `selected`).
2. Реализовать очистку временной таблицы при новом промте.
3. Реализовать получение строк с `selected = True` для сохранения.

---

## Этап 5. Графический интерфейс (`main.py`)

### 5.1. Главное окно

1. Разделить окно на зоны: ввод промта, таблица результатов, панель действий.
2. Добавить поле ввода промта и кнопку «Отправить».
3. Добавить выпадающий список или боковую панель для выбора сохранённых промтов из БД.

### 5.2. Таблица результатов

1. Отобразить временную таблицу: колонки «Модель», «Ответ», «Выбрать» (чекбокс).
2. Заполнять таблицу после получения ответов от всех активных моделей.
3. Показывать индикатор загрузки на время запросов.

### 5.3. Действия пользователя

1. Кнопка «Сохранить» — записать отмеченные строки в `results`, очистить временную таблицу.
2. Кнопка «Новый запрос» — очистить поле промта и временную таблицу.
3. Обработка ошибок сети: вывод сообщения пользователю, строка с текстом ошибки в таблице.

---

## Этап 6. Управление моделями и промтами (GUI)

1. Добавить окно или вкладку «Модели»: список, добавление, редактирование, включение/отключение (`is_active`).
2. Добавить окно или вкладку «Промты»: просмотр сохранённых промтов, теги, выбор для повторной отправки.
3. Добавить окно «История результатов»: просмотр сохранённых записей из таблицы `results`.

---

## Этап 7. Настройки

1. Реализовать чтение/запись настроек через таблицу `settings` (путь к БД, таймаут запросов, тема и т.д.).
2. Добавить экран настроек в GUI (минимальный набор параметров).

---

## Этап 8. Дополнительные функции (опционально)

1. Поиск и сортировка во всех таблицах GUI.
2. Экспорт выбранных результатов в Markdown и JSON.
3. Логирование запросов в файл (`logs/`).
4. Сборка исполняемого файла через PyInstaller.

---

## Этап 9. Тестирование и финализация

1. Проверить полный сценарий: ввод промта → отправка → выбор строк → сохранение → просмотр истории.
2. Проверить работу с неактивными моделями и отсутствующими API-ключами.
3. Проверить повторный выбор сохранённого промта.
4. Обновить `README.md`: описание установки, настройки `.env`, запуск.

---
